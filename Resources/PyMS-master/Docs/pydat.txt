PyDAT v1.12
PyDAT is used to convert DAT (units.dat, weapons.dat, flingy.dat, sprites.dat, images.dat, upgrades.dat, techdata.dat, sfxdata.dat, portdata.dat, mapdata.dat, orders.dat) to and from TXT files.

	Syntax	syntax
Every decompiled DAT file uses the same syntax, just with different keywords. Each entry starts with an ID header, then is followed by a list of all the settings for that id:
<Code
TypeID ID:
    Data	Value
    Data	Value
    ...
>
TypeID and the Data/Value pairs depend on which type of DAT file it is. The different types are:
 +u
 -[#unitdata	UnitID]
 -[#weapondata	WeaponID]
 -[#flingydata	FlingyID]
 -[#spritedata	SpriteID]
 -[#imagedata	ImageID]
 -[#upgradedata	UpgradeID]
 -[#techdata	TechID]
 -[#sounddata	SoundID]
 -[#portraitdata	PortraitID]
 -[#mapdata	MapID]
 -[#orderdata	OrderID]
+u
ID is the ID of the entry in the DAT file (for example UnitID 0 is Terran Marine). There are only three types of values in a DAT file:
 +u
 -Quantity (like MineralCost or BuildTime)
 -Flags (like SpecialAbilityFlags)
 -Data entry (either from another DAT file, TBL file, or one of a list of values, given in the [#reference	Reference])
+u
When decompiling entries from a DAT file, data entry values will always have a comment at the end, telling which type of data entry it is and giving additional information on the value. The only values that are flag sets are the values used by data keywords ending in Flags (like SpecialAbilityFlags). Everything else is a quantity.
		Flags	flags
The only entries that take flags are SpecialAbilityFlags, StarEditGroupFlags, and StarEditAvailabilityFlags in units.dat, and TargetFlags in weapons.dat. The value for Flags is a series of zero's and one's, stating which settings are on or off (zero's meaning the flag is off, one's mean its on). SpecialAbilityFlags has 32 flags, in the following order:
 +o
 -Building
 -Addon
 -Flyer
 -Worker
 -Subunit
 -Flying Building
 -Hero
 -Regenerates HP
 -Animated Idle(?)
 -Cloakable
 -Two Units in 1 Egg
 -Single Entity
 -Resource Depot
 -Resource Container
 -Robotic Unit
 -Detector
 -Organic Unit
 -Requires Creep
 -Unused(?)
 -Requires Psi
 -Burrowable
 -Spellcaster
 -Permanent Cloak
 -Pickup Item(?)
 -Ignore Supply Check
 -Use Medium Overlays
 -Use Large Overlays
 -Battle Reactions
 -Full Auto-Attack
 -Invincible
 -Mechanical Unit
 -Produces Units(?)
+o
StarEditGroupFlags takes 8 flags in the following order:
 +o
 -Zerg
 -Terran
 -Protoss
 -Men
 -Building
 -Factory
 -Independent
 -Neutral
+o
StarEditAvailabilityFlags takes 16 flags in the following order:
 +o
 -Non-Neutral
 -Unit Listing and Palette
 -Mission Briefing
 -Player Settings
 -All Races
 -Set Doodad State
 -Non-Location Triggers
 -Unit and Hero Settings
 -Location Triggers
 -BroodWar Only
 -Unused (0x400)
 -Unused (0x800)
 -Unused (0x1000)
 -Unused (0x2000)
 -Unused (0x4000)
 -Unused (0x8000)
+o
And TargetFlags takes 9 flags in the following order:
 +o
 -Air
 -Ground
 -Mechanical
 -Organic
 -Non-Building
 -Non-Robotic
 -Terrain
 -Organic or Mechanical
 -Own
+o
		Restrictions	restrictions
Some data entries are restricted to certain ID's in the DAT files, they are:
 +u
 -units.dat entries 106-201 are the __only__ unit entries that have the data entries: Infestation, AddonHorizontal, AddonVertical
 -units.dat entries 0-105 are the __only__ unit entries that have the data entries: ReadySound, PissSoundStart, PissSoundEnd, YesSoundStart, YesSoundEnd
 -sprites.dat entries 130-516 re the __only__ sprite entries that have the data entries: HealthBar, SelectionCircleImage, SelectionCircleOffset

	Example Usage	examples
Decompiling unit id 99 (Samir Duran), then recompiling back into the same dat file:
<Command Line
python PyDAT.pyw -n 99 units.dat
  ..edit dat..
python PyDAT.pyw -b units.dat -c units.txt
>
Decmpiling every entry in mapdata.dat, then recompiling:
<Command Line
python PyDAT.pyw -m MyMod\mapdata.dat
  ..edit dat..
python PyDAT.pyw -m -c MyMod\mapdata.dat
>
